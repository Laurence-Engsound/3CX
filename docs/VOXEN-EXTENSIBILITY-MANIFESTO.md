## 為什麼這份文件存在

VOXEN 對外的承諾不是「我們已經做了 70 個模組」，而是「**我們做的這 70 個模組，是站在一個能再長 700 個的骨架上**」。

本文件用一個**未來情境**證明這個承諾：當客戶提出「**門市銷售員配 AI 眼鏡，做即時銷售輔助 + 事後質檢**」這個今天還沒人做、明天會有人做的需求時，VOXEN 架構**不需要重寫**就能容納。

---

## 三句話講完 VOXEN 架構

VOXEN 是 **6 + 6 + 4** 的分層平台：

- **6 層主架構**（L1 整合 → L6 體驗）：垂直堆疊，上層呼叫下層，事件反向流
- **6 橫切柱**（C1 資安 → C6 國際化）：穿透所有層的通用關注點
- **4 平台營運**（P1 開發者 → P4 合作夥伴）：支撐這個產品成為一門生意

每個 ▢ 都有清楚的契約。每個契約都是 **open enumeration**（開放式分類）— 已知必要先列，未來可加。

## 擴展性的五個明文承諾

VOXEN 對客戶 / 投資者 / 合作夥伴的明文擴展承諾：

### 承諾 1 — L1 整合層的 Adapter 群是開放的

目前 9 群（PBX / CRM / AI / Identity / Messaging / Email / Data Export / Financial），不是上限。

**未來可加**：Edge Device、IoT、Logistics、Industry-specific（醫療 HIS / 銀行核心系統）、新國家在地服務。

加新群**不影響任何上層** — Adapter 內部處理協定差異，上層只看統一介面。

### 承諾 2 — L3 互動核心的 Channel 不限通訊距離

目前 8 channels（Voice / Chat / Email / SMS / WhatsApp / Line / Social / Video）— 全是「遠距通訊」。

**Channel 的真正定義**：「客戶與企業之間任何一段可被識別的互動空間，**不限通訊距離與媒介**」。

**未來可加**：實體門市互動、AR/VR 元宇宙互動、車載互動、智能家居互動。每一種都用相同的 IChannel 介面接入。

### 承諾 3 — L4 智能層是多模態的

目前模組偏 NLP（語音 / 文字）— 因為這是客服中心當下的核心。

**L4 的本質定義**：「**多模態智能服務 — 涵蓋語音 / 語言 / 視覺 / 預測 / 合規 / 輔助等所有 AI 能力**」。

**未來可加**：Computer Vision 群（物件偵測 / 動作識別 / 場景理解 / AR 渲染決策）、生物資訊群（生物辨識 / 健康監控）、空間智能群（定位 / 動線分析）。

L4 的 AI Service Connector 就是設計來吸納任何 AI 供應商的，不限 NLP。

### 承諾 4 — L6 體驗層的角色不限「桌前」

目前角色偏「**坐辦公桌的內部營運者**」（座席 / 主管 / IT / 稽核）+ 客戶端自助。

**L6 的真正定義**：「**任何使用 VOXEN 完成工作的人之介面**」。

**未來新角色類別**：**Field Worker**（現場工作者） — 戴 AI 眼鏡的門市銷售員、配備 AR 頭盔的維修技師、車載終端的物流司機、急救現場的醫護人員。

新角色不影響現有角色 — 各自獨立的入口模組（FM-L6-AGENT / FM-L6-SUP / FM-L6-FIELD-* / 等）。

### 承諾 5 — 橫切柱可被擴充

目前 6 柱（Security / Governance / Observability / Multi-tenancy / DevOps / Global Readiness）— 涵蓋當下平台級必要關注點。

**未來可加 C7+**：例如 **C7 Sustainability**（綠色運算 / 碳足跡）、**C8 Accessibility Beyond a11y**（無障礙的進階）、**C9 AI Ethics Governance Deep**（演算法問責）。

加新柱需 ARB（架構評審委員會）通過，但**架構不擋路**。

## 證明題：AI 眼鏡情境是否能放進現有架構？

讓我用一個**今天還沒人做、未來必會做**的情境壓力測試。

**情境**：高端零售商客戶要求 — 門市銷售員配戴 AI 眼鏡，銷售過程中：
- 攝像頭即時辨識客戶所看的商品
- AI 即時推薦話術 / 商品搭配 / 庫存資訊
- AR 將額外資訊投影給客戶看（例如商品材質、原產地、保養方式）
- 全程錄影 + 錄音
- 事後 AI 自動質檢評分（招呼語、商品介紹完整度、結帳禮貌、合規話術）

這個情境融合 **邊緣設備 + 多模態 AI + AR 互動 + 質檢工作流**。對照 VOXEN 架構：

| 需要的能力 | VOXEN 對應 | 修補成本 |
|:--|:--|:--:|
| AI 眼鏡 SDK 整合 | L1 新增 `FM-L1-EDGE-{品牌}` Adapter | **低** — 加新 Adapter 群即可 |
| 視訊 / 音訊串流接收 | L3 已有 Media Pipeline + Video Channel | **零** — 既有能力延伸 |
| 客戶資料統一視圖 | L2 Customer 360 已 channel-agnostic | **零** — 加新 channel 值即可 |
| 商品 / 話術知識庫 | L2 Knowledge Base | **零** — 既有能力 |
| Computer Vision 處理 | L4 新增 `FM-L4-VISION-*` 群 | **中** — 新模組群 |
| 即時話術建議 | L4 已有 `FM-L4-COACH` | **零** — 既有能力 |
| AR 內容渲染決策 | L4 新增 `FM-L4-VISION-AR` | **中** — 屬視覺群 |
| 銷售流程編排 | L5 Flow Designer + Rules Engine | **零** — 既有能力 |
| 跨 channel 客戶旅程 | L5 Journey Orchestrator | **零** — 既有能力 |
| 銷售員端體驗 | L6 新增 `FM-L6-FIELD-COMPANION`（新角色類別）| **中** — 新體驗模組 |
| 事後質檢評分 | L6 QM + L4 新增 AutoScore | **低-中** — 加 AI 評分模組 |
| 影像錄製治理 | C2 新增「Bystander 隱私」「Worker 監控倫理」子節 | **低** — 加治理規範 |
| 設備生命週期 | 未來 P5 Edge Device Operations | **中** — 新營運塊 |

**結論**：13 項能力 × **零項需要動現有層級或柱的核心結構**。所有擴展都在預設好的「擴展點」內。

**修補總成本**：約 4-6 個新模組 + 2-3 個治理子節 + 預留新營運塊。**架構零打掉重練**。

## 為什麼這個證明題重要

如果同一個情境拿給「**單體產品**」型客服軟體（不論國內外大廠），會發生什麼？

- 「**這不是我們的應用範圍**」 — 因為他們的範圍由功能清單定義
- 「**需要客製化開發**」 — 客製等於分叉，下次升級全炸
- 「**6 個月後再評估**」 — 因為要重新設計資料模型 / API / 部署
- 「**請等我們的下一代產品**」 — 等於 2-3 年後，客戶等不起

VOXEN 的回應應該是：

> 「現在沒做。但我們的架構**已經為此預留了位置**。
>  L1 加 Edge Adapter 群、L4 加 Vision 模組、L6 加 Field Worker — 各自獨立，不影響其他客戶。
>  共建 PoC 6-12 週可走通；2026 Q4 / 2027 Q1 上 production。」

差距在哪？**差在架構是否預先承認自己會被未來改變**。

VOXEN SRS v2.0 的整個設計哲學就是：**承認自己有限，但設計時為無限留路**。

## 給內部團隊的紀律

這份宣言對內部開發者有四條**不能違反的紀律**：

1. **不在現有層內塞「特殊邏輯」** — 看到「if 是 AI 眼鏡情境就...」的 `if` 應該觸發 ARB 警報
2. **新需求先問「這該在哪一層」** — 答不出來表示要寫 ADR
3. **新模組必須符合該層的契約介面** — 不允許「特例介面」
4. **加新層 / 新柱必經 ARB**（已寫於 Ch 7.X 治理規則）

## 給客戶的承諾

當客戶問「VOXEN 能不能做 X？」時，標準答覆框架：

```
1. 「X 在我們架構的 [L?] 層」
2. 「對應的擴展點是 [明確的擴展類別]」
3. 「我們現在 [已做 / 規劃中 / 願景]」
4. 「如果您需要，6-12 週可以 PoC，整體不影響其他模組」
```

如果四條都答得出來 — VOXEN 是平台。
任何一條答不出來 — 我們欠客戶一個 ADR。

## 給投資者 / 合作夥伴的承諾

VOXEN 的 TAM（潛在市場規模）不是「客服中心軟體」（台灣 ~30 億 / 年），而是 **「企業客戶互動平台」**（台灣 ~120 億 / 年，加上門市 / 醫療 / 金融現場 / 物流現場 / 政府窗口等場景）。

這個 TAM 不是靠「賣更多模組」達成 — 是靠「**架構讓我們能去任何客戶互動發生的地方**」達成。

本文件就是這個論述的**架構證據**。
