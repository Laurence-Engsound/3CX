# E.SUN Outreach Project (玉山 Phase 6) — 會後 24 小時 Follow-up Email 模板

**用途**：玉山會議結束後 24 小時內必發。給玉山「我們動作快、組織力強」的訊號，並把對話轉成書面紀錄。

**寄發對象**：
- TO：玉山主談人（PM / 採購 / IT 主管）
- CC：玉山會議參與者全部
- BCC：瑛聲內部（Engagement Manager、業務主管、技術 Lead）

**寄發時機**：會議結束後 4-6 小時內最佳；最遲 24 小時。

## 撰寫提醒清單（寄出前 self-check）

寄出前 5 分鐘做最後檢查：

### 內容正確性
- [ ] 玉山方 / 瑛聲方參與者姓名拼寫正確
- [ ] 所有日期與時間都已從相對日期（「下週」「兩週後」）轉為絕對日期（5/XX）
- [ ] Action Items 的「負責方」明確標出（不要寫「雙方共同」）
- [ ] Action Items 的「期限」具體到日（不要只寫「儘快」）
- [ ] 敏感數字（席次數、預算範圍）已對齊會議現場確認的版本

### 附件管理
- [ ] PDF 附件全部是「加完浮水印」的版本，不是無水印的源檔
- [ ] 內部專用文件（Q&A Binder、TCO 試算表）**絕對不要**夾帶
- [ ] 附件總大小 < 25 MB（避免被玉山 Mail Server 退信）；超過用瑛聲 share link

### 語氣與格式
- [ ] 不使用「我們將」「我們會努力」這類弱動詞，改用「瑛聲承諾於 X 日前提供 Y」
- [ ] 不過度承諾（「保證」「絕對」字眼避免）
- [ ] 不批評其他競品或 Genesys 原廠
- [ ] 簽名檔含手機（玉山可即時聯絡）

### 寄發前
- [ ] CC 名單完整（玉山方所有與會者 + 瑛聲方關係人）
- [ ] BCC 不要把玉山主管放錯位置（曾發生 BCC 誤打 CC 的事故）
- [ ] 重要附件「點開檢查」一次（避免附錯版本）
- [ ] 主旨欄無錯字，且包含案名識別碼（「E.SUN Outreach Project (玉山 Phase 6)」）

## 三種變體小貼士

**變體 A 用法（順利）**：直接發本模板，重點放在 Action Items 與下一次會議。

**變體 B 用法（需補資料）**：在「下一次會議建議」段之前加一段「會議中關於 [X] 議題的補充說明」，把當天現場沒講完整的內容書面補齊。

**變體 C 用法（有疑慮）**：在「會議重點結論」第 1 點之前先加一段「會議重點疑慮回應」，逐項回應玉山當天提出的擔憂，並附上對應的證據檔案。最後一段語氣放軟，強調「願意配合玉山評估節奏」，不要逼迫快速決定。
